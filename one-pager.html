Perseus & Mimir: Open-Source Infrastructure
for AI Control & Interpretability

Technical capability brief — Thomas Connally · tcconnally@gmail.com · github.com/tcconnally
Two production-deployed, MIT-licensed systems that directly address the AI Forge Critical AI Challenges. Perseus grounds LLMs with verified context at every decision boundary (AI Control). Mimir provides auditable persistent memory with full provenance trails (AI Interpretability). Both are self-contained sidecar processes — no cloud dependencies, deployable on air-gapped infrastructure. Integrated with agent platforms today.

Perseus · Live Context Engine

AI ControlChallenge 2Challenge 4

Pre-resolves environment state at every session boundary and injects a canonical snapshot before the model acts. Eliminates context decay — the dominant failure mode in long-running agentic AI.

→ Challenge 4 (Agentic Interpretability): Every context snapshot is a verifiable decision point. Operators trace any agent action to the exact state that informed it — system-level auditing without model internals.
→ Challenge 2 (Long-Horizon Failures): Snapshot timeline enables replay and root-cause isolation. Investigators identify when drift entered and which decisions it corrupted.
Status: Live · License: MIT · Footprint: Single process, no GPU · Integration: Hermes Agent platform · perseus.observer

Mimir · Auditable Memory Engine

AI InterpretabilityChallenge 2Challenge 4

Stores structured entities, insights, and architectural decisions with full provenance trails. Unlike opaque vector stores, every memory operation is typed, timestamped, and logged.

→ Challenge 2 (Verifiable Accountability): Audit trails make black-box failure analysis forensic. Trace exactly what the system knew, when it learned it, and when contamination entered — no speculation.
→ Challenge 4 (Agentic Interpretability): 23 typed MCP tools expose the agent's full mental model. Operators see what it believes, why, and how beliefs evolved across long deployments.
Status: v0.5-rc · License: MIT · Backend: SQLite, no external deps · Tools: 23 MCP tools, 40+ entities · github.com/tcconnally/mimir
